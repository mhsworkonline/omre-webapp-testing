# CLAUDE.md — omre-webapp Test Suite

## Project Overview
End-to-end Playwright test suite for **app.omre.ai** — a social media / all-in-one platform (OMRE). Tests cover the full surface area of the web app across multiple feature domains.

## Tech Stack
- **Test framework:** Playwright v1.60 (`@playwright/test`)
- **Language:** JavaScript (ESM — `import` syntax)
- **Auth:** `dotenv` + stored session via `playwright/.auth/user.json`
- **Browsers:** Chromium, Firefox, WebKit (primary runs use `--project=chromium`)
- **Reports:** Timestamped HTML / JSON / JUnit in `playwright-report/` and `test-results/`

## AI Coding Methodology (Claude Code)
- **Batch-read before fixing** — always read all relevant files in one parallel pass before making any edits; never read-then-fix one file at a time
- **Minimal tokens** — no explanatory prose unless explicitly asked; output diffs/commands/results only
- **Parallel edits** — apply all fixes in a single message where files are independent
- **Skip guards over hard assertions** — all optional UI checks must use `isVisible().catch(() => false)` + `test.skip(); return;` pattern before `expect(...).toBeVisible()`
- **Never explain tool calls** — tool usage is not visible to user; only final results matter

## Running Tests

### Full suite (with parallel workers)
```powershell
npx playwright test --project=chromium --workers=4
```

### Only previously failing tests (use --grep, not line numbers)
```powershell
powershell -File "C:\claude-folder\webapp-testing\omre-webapp\run-failing-tests.ps1"
```

### Single file
```powershell
npx playwright test tests/home/home.spec.js --project=chromium
```

### Auth setup (run first if session is expired)
```powershell
npx playwright test tests/auth/auth.setup.js --project=auth
```

### Rename test descriptions to Given-When-Then
```powershell
node rename-tests.js
```

## Key Conventions
- All spec files use `test.use({ storageState: 'playwright/.auth/user.json' })` for authenticated sessions
- `test.setTimeout(45000)` is the global timeout; slow describe blocks (e.g. TC-BIZ Business Detail) use `test.setTimeout(90000)`
- **Never use `waitUntil: 'load'` or `waitUntil: 'networkidle'`** — the SPA has constant background polling; always use `waitUntil: 'domcontentloaded'` followed by `waitForTimeout(1500)`
- Nav selectors: use `nav, [role="navigation"], aside` — the app does not always render a `<nav>` HTML element
- Main content: use `main, [role="main"], body > div:not([hidden])` — some pages omit `<main>` and the first `body > div` is often a hidden element
- Search inputs: restrict to `input[...]` / `textarea[...]` — `[aria-label*="search"]` can match non-input wrappers
- Page title: app uses `"Omre"` as the generic title; match with `/omre|feature-name/i`
- **Never use `.or()` chains for assertions** — Playwright strict mode requires exactly one element; use sequential `isVisible()` checks instead
- **All hard `expect(...).toBeVisible()` calls on optional UI** must have a preceding `isVisible()` skip guard

## Code Conventions
- All spec files are **plain JavaScript (ESM)** — never use TypeScript syntax: no `type annotations`, no `!` non-null assertions, no `as Type` casts, no `Promise<T>` generics, no `: string[]` array types
- Inside `page.evaluate()` callbacks, access non-standard performance entry properties directly (e.g. `entry.value || 0`) — no TypeScript casts

## Known App Behaviours
- **React hydration error #418** fires on notifications/messages pages — filtered out in error boundary tests
- **HTTP 502 errors** from Cloudflare are transient — re-run affected tests rather than treating as test bugs
- **Auth sessions expire** during very long runs — re-run `auth.setup.js` before a fresh full suite run
- **Platform hub pages** (`/biz`, `/learn`, `/link`) show landing pages with div-based cards, not `<li>` or `<article>` — listing tests use skip guards
- **`body > div` is always hidden** on channel, subscriptions, messages, and AI Studio pages — always use `body > div:not([hidden])`
- **Next.js RSC payload** injects `"children":404` into every page's `<script>` tags — use `innerText()` not `textContent()` to avoid false-positive 404 matches
- **`.tap()` not supported** in Chromium project (no `hasTouch: true`) — always use `.click()` instead
- **`fill()` on `input[type="number"]`** rejects non-numeric text — use `pressSequentially()` instead
- The app title is always `"Omre"` regardless of the current page/feature
